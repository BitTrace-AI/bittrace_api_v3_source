# BitTrace API v3 Hardening Release (package 0.3.1)

## Summary

This release is a stabilization pass on the current BitTrace API v3 source
repository. It does not redesign the product, expand the supported lane, or
change the stable public CLI or import surface.

## Stable Public Surface

- import namespace: `bittrace`
- CLI: `bittrace`
- module fallback: `python -m bittrace`
- stable commands:
  - `bittrace campaign`
  - `bittrace verify`
  - `bittrace deployment-candidate`
  - `bittrace persistence`
- experimental boundary:
  - `bittrace experimental ...`
  - `bittrace.experimental`
  - `configs/experimental/`

## What Changed

- Added a visible `tests/` tree with pytest coverage for CLI help, module
  imports, and stable config/schema loading.
- Added a lightweight `scripts/run_ci_smoke.py` path for compile/import/help
  smoke in CI-safe environments.
- Added GitHub Actions CI for `push` and `pull_request` on Python 3.12 with
  editable install and automated checks.
- Added release hygiene files: `CHANGELOG.md`, `KNOWN_ISSUES.md`, a release
  checklist, a concise PR template, and reusable PR/release markdown.
- Clarified the version story:
  `BitTrace API v3` is the repository and product generation, while
  `bittrace-api-v3-source` package version `0.3.1` is a distribution release
  within that same v3 generation.
- Cleaned the README documentation table and removed the stray `Need | Home`
  wording.

## Verification And Quality Gates

- Automated checks:
  - `python -m pytest`
  - `python scripts/run_ci_smoke.py`
- Manual release gate:
  - `python scripts/run_release_smoke.py`
- The full release smoke remains manual because it requires the isolated
  `.venv_source`, the `.[gpu]` extra, and the repo-local stable workflow path.

## Known Limitations

- The shipped canonical source profile still points at a repo-external raw
  dataset root.
- The full release smoke remains heavier than CI and is not a generic hosted
  runner check.
- Experimental commands remain intentionally unstable and outside compatibility
  guarantees.
- The supported public lane remains narrow by design.

## Upgrade / Compatibility Notes

- This release preserves the stable v3 public CLI/import surface.
- Existing `bittrace` imports and the stable top-level commands remain valid.
- Experimental commands remain under `bittrace experimental ...` and are still
  intentionally outside the supported commercial lane.
- The package version advances from `0.3.0` to `0.3.1` without changing the v3
  public surface.
